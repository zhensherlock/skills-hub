# 猎头职位验证

企查查 MCP 服务 - 专注职位真实性与企业核实

## ⚙️ 配置要求

> **如果已有 MCP 配置**（如在 CodeBuddy 或其他 IDE 中），无需重复配置，可直接使用工具。

### 获取 API Key

1. 访问 https://agent.qcc.com 登录/注册
2. 在控制台获取个人 API Key
3. 配置环境变量 `QCC_AGENT_API_KEY`

> **如果用户未提供 API Key**，必须先引导用户获取，否则所有工具调用将返回鉴权失败。

## 快速开始（首次使用必读）

首次使用前，运行 setup.sh 完成 MCP 服务注册：

```bash
bash setup.sh
```

### 验证配置

```bash
mcporter list | grep -E "(company-server|risk-server|operation-server)"
```

## ⚠️ 强制工作流程

### 1. 检查 MCP 配置

在调用任何工具前，**必须**先确认 MCP server 已正确配置：

```bash
mcporter list | grep -E "(company-server|risk-server|operation-server)"
```

如未找到，引导用户运行 `bash setup.sh` 完成配置。

### 2. API Key 验证

首次调用工具前，验证 API Key 是否有效（通过调用任意工具测试）：

```bash
mcporter call company-server get_company_registration_info --args '{"searchKey": "测试企业"}'
```

如返回鉴权失败，引导用户重新获取 API Key。

### 3. 查询策略（分级查询）

**Tier 1 - 必查项（7 个核心工具）：**

**企业真实性核实（优先执行）：**
- `get_company_registration_info` (company-server) — 企业工商信息（企业存续）
- `get_dishonest_info` (risk-server) — 失信信息（信用风险）
- `get_business_exception` (risk-server) — 经营异常（经营异常）

**融资与规模：**
- `get_financing_records` (operation-server) — 融资情况（资金实力）
- `get_annual_reports` (company-server) — 企业年报（公司规模）

**联系方式核实：**
- `get_contact_info` (company-server) — 联系方式（地址电话核实）

**基础风险：**
- `get_judgment_debtor_info` (risk-server) — 被执行人（债务风险）

**决策点：** 如企业不存在/已注销/失信，立即向用户预警，可能是虚假职位。

**Tier 2 - 扩展查询（根据用户需求选择）：**
- `get_shareholder_info` (company-server) — 股东信息（股东背景）
- `get_key_personnel` (company-server) — 主要人员（核心团队）
- `get_serious_violation` (risk-server) — 严重违法（重大违规）
- `get_administrative_penalty` (risk-server) — 行政处罚（合规风险）
- `get_news_sentiment` (operation-server) — 新闻舆情（负面新闻）

### 风险等级判定标准

| 风险等级 | 判定标准 | 职位建议 |
|---------|---------|---------|
| **红色风险 ⛔** | 企业不存在、注销/吊销、失信被执行（有任何记录）、严重违法（有任何记录）、经营异常（地址失联） | 高危，可能是虚假职位 |
| **橙色风险 ⚠️** | 经营异常（未年报/已移出）、被执行（涉案金额≥50 万）、行政处罚（2 年内≥2 次）、联系方式不匹配 | 谨慎，需进一步核实 |
| **黄色风险 ⚡** | 少量处罚（单笔<10 万）、规模较小（参保人数<10）、成立时间短（<1 年） | 一般关注，可正常面试 |

### 工具判定逻辑

#### get_company_registration_info 判定
```
红色风险：登记状态 包含"注销"、"吊销"、"清算"
重点关注：登记状态、成立日期、注册资本
```

#### get_dishonest_info 判定
```
红色风险：失信信息[] 数组有任何记录
重点关注：涉案金额、立案日期、发布日期（最近一次）
```

#### get_business_exception 判定
```
红色风险：列入经营异常名录原因 包含"地址失联"
橙色风险：列入经营异常名录原因 为"未年报"（无论是否移出）
重点关注：列入日期（最近一次）、列入原因
```

#### get_judgment_debtor_info 判定
```
橙色风险：执行标的≥500000
红色风险：执行标的≥1000000
重点关注：执行标的、案件状态（执行中）
```

#### get_administrative_penalty 判定
```
橙色风险：处罚金额≥50000 或 2 年内处罚次数≥2 次
黄色风险：处罚金额<50000 且 2 年内仅 1 次
重点关注：处罚金额、处罚日期、处罚结果
```

#### get_contact_info 判定
```
核实要点：
- 面试地址 vs 注册地址 是否一致
- HR 联系电话 vs 企业电话 是否匹配
- 邮箱是否为企业邮箱（非 QQ/163 等免费邮箱）
- 是否有官方网站
```

### 响应处理规范

- **数据时效性**：所有报告必须标注数据截至日期
- **真实性优先**：企业真实性必须在首屏展示
- **建议明确**：必须给出真实/可疑/虚假的明确结论

## 触发场景

### 必须使用此技能的场景

- 猎头推荐职位后核实企业
- 收到陌生 HR 联系后验证
- 面试前确认公司真实性
- 核实职位 JD 中的企业信息
- 验证创业公司/小微企业
- 跨城市求职远程面试前

### 触发关键词

猎头、职位验证、公司真假、HR 联系、面试邀请、创业公司、小微企业、靠谱吗、是真的吗

### 不触发边界

- 用户需要**求职公司调查**（应使用 job-company-check 技能）
- 用户需要**供应商审查**（应使用 supplier-audit 技能）
- 用户需要**投资尽调**（应使用 investment-dd 技能）
- 用户进行**文档编辑、日程管理**（非企业背景调查）

## 工具使用

### 0. 基础工具

#### `get_company_registration_info` — 企业工商信息

**用途**：查询企业核心登记信息，验证企业身份

**参数**: 
- `searchKey`(必填) - 企业名称或统一社会信用代码

**返回关键字段**：
- 企业名称 - 企业名称
- 统一社会信用代码 - 统一社会信用代码
- 法定代表人 - 法定代表人
- 登记状态 - 登记状态（在业/存续为正常，注销/吊销为异常）
- 成立日期 - 成立日期
- 注册资本 - 注册资本
- 实缴资本 - 实缴资本
- 企业类型 - 企业类型
- 核准日期 - 核准日期
- 注册地址 - 注册地址
- 经营范围 - 经营范围
- 参保人数 - 参保人数
- 分支机构参保人数 - 分支机构参保人数

> ⚠️ **登记状态判断**：
> - 在业/存续 → 正常
> - 注销/吊销/清算 → 红色风险（企业已不存在）
> 📊 **规模判断**：
> - 参保人数≥100 人 → 规模较大
> - 参保人数 10-100 人 → 中等规模
> - 参保人数<10 人 → 小微企业
> - 参保人数为 0 → 需警惕（可能是空壳）

### 1. 真实性核实工具（核心）

#### `get_dishonest_info` — 失信信息 ⛔

**用途**：查询企业是否被列入失信被执行人名单

**参数**: 
- `searchKey`(必填) - 企业名称或统一社会信用代码

**返回关键字段**：
- 企业名称 - 被查询企业名称
- 摘要 - 记录总数和涉案总金额汇总
- 关联分析 - 关联风险提示（限高/被执行等）
- 失信信息[] - 失信明细列表
  - 案号 - 执行案号
  - 失信被执行人名称 - 失信被执行人名称
  - 涉案金额 - 涉案金额（元）
  - 执行法院 - 执行法院
  - 立案日期 - 立案日期
  - 发布日期 - 发布日期

> ⚠️ **一票否决**：有任何当前有效的失信记录，直接列为红色风险，可能是虚假/高风险职位

#### `get_business_exception` — 经营异常 ⚠️

**用途**：查询企业是否被列入经营异常名录

**参数**: 
- `searchKey`(必填) - 企业名称或统一社会信用代码

**返回关键字段**：
- 企业名称 - 被查询企业名称
- 摘要 - 记录总数和最近列入日期
- 经营异常信息[] - 经营异常明细列表
  - 列入日期 - 列入日期
  - 作出决定机关 (列入) - 作出决定机关
  - 列入经营异常名录原因 - 列入原因（未年报/地址失联等）

> ⚠️ **风险判定**：
> - 地址失联 → 红色风险（可能是皮包公司）
> - 未年报（已移出） → 橙色风险
> - 未年报（未移出） → 橙色风险

---

#### `get_judgment_debtor_info` — 被执行人 ⚠️

**用途**：查询企业作为被执行人的案件信息

- `searchKey`(必填) - 企业名称或统一社会信用代码

> ⚠️ **风险判定**：
> - 执行中金额≥50 万 → 橙色风险（资金链可能有问题）
> - 执行中金额≥100 万 → 红色风险

## 一、企业基本信息

- **法定代表人**：XXX
- **注册资本**：XXX

### 2.1 企业存续状态
- 登记状态：[存续/在业/注销/吊销]
- 成立时间：[X 年]
- 核实结果：[真实/可疑/虚假]

### 2.2 失信记录
- 失信被执行人：[有/无]
- 详情：[如有，列出]
